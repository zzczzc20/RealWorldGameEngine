SVM_net 坑口活动: 用户体验旅程与关键洞察阶段

1. 触达与预期

观察用户行为:

  • 如何发现活动？
  • 对宣传信息的反应？
  • 报名动机与决策过程。

初步信息收集点:

  • 用户的初始兴趣点？
  • 对“现实AI游戏”的理解？
  • 参与活动的期望是什么？

方法: 宣传分析, 报名问卷

→
2. 启动与探索

观察用户行为:

  • App上手与导航。
  • 进入“坑口”场景的初步反应。
  • 如何开始寻找线索/任务？

初步信息收集点:

  • App易用性感知？
  • 对游戏世界观的代入感？
  • 早期遇到的困惑点？

方法: App日志, 初期观察

→
3. 核心互动 & 解谜

观察用户行为:

  • 与物理环境线索的互动。
  • 谜题解决策略与尝试。
  • 与AI NPC的对话模式。
  • 遇到卡点时的反应。

初步信息收集点:

  • 对AI智能程度的感知？
  • 谜题的趣味性与挑战性？
  • 现实与数字结合的流畅度？
  • “Aha!”时刻何时出现？

方法: 现场观察(AEIOU), App日志, 即时提问

→
4. 反馈与分享

观察用户行为:

  • 活动结束后的即时情绪。
  • 是否主动讨论/分享？
  • 对体验的总体评价。

初步信息收集点:

  • 最深刻的体验点？
  • 未被满足的期待？
  • 对AI和现实游戏的看法有无改变？

方法: 结束访谈, 问卷
